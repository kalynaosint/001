# 内容协调投票系统

为小组的 Bilibili / 抖音 / 知乎 三个账号提供内容发布协调投票。

## ⚠️ 关于数据存储的重要说明

**当前版本使用浏览器 `localStorage` 存储数据。这意味着：**

- ✅ 完全免费，无需后端
- ✅ 可以直接部署到 GitHub Pages
- ❌ **每个人在自己的电脑/浏览器上看到的数据是独立的，A 投的票 B 看不到**
- ❌ **换浏览器/换设备/清缓存数据就会丢失**

也就是说，**当前版本无法实现真正的多人协作投票**，只能作为本地原型/演示。

### 升级到云端存储的建议

要真正让小组协作，请将 `app.js` 中的 `loadRequests()` / `saveRequests()` 替换为云端调用。推荐方案：

1. **Firebase Realtime Database**（免费额度足够，10 行代码改动）
2. **Supabase**（开源，免费层大方）
3. **GitHub Issues API** + Personal Access Token（最 hack，但完全免费）

如果你需要其中任一方案的迁移代码，告诉我，我可以补上。

## 部署步骤（GitHub Pages）

1. **编辑 `config.js`** — 填入小组真实成员姓名和密码
   - 3 个运营者的 `role` 分别设为 `operator_bilibili` / `operator_douyin` / `operator_zhihu`
   - 17 个普通组员的 `role` 设为 `member`
2. 把整个文件夹推到 GitHub 仓库（例如 `voting-site`）
3. 仓库 Settings → Pages → Source 选择 `main` 分支根目录 → Save
4. 几分钟后访问 `https://<你的用户名>.github.io/voting-site/`

## 投票规则

### 加权机制

| 角色 | 权重 |
|------|------|
| 内容作者本人 | **3**（自动同意，无需投票） |
| 组长 | **3**（对所有请求生效） |
| 所涉平台的运营者 | **3**（仅对涉及自己平台的步骤生效） |
| 其他运营者 / 普通组员 | **1** |

> "所涉平台运营者"指：对于"主发布"步骤，作者负责平台的运营者；对于"同步到 X 平台"步骤，X 平台的运营者。
> 组长对**所有**步骤都享受权重 3，无论涉及哪个平台。

### 通过条件

每个请求拆分为多个独立投票步骤：
- **主发布步骤**：作者发到自己负责的平台，是否合适（**提交后 1 小时截止**）
- **同步步骤**（每个目标平台一个）：是否同意同步过去（**提交后 3 小时截止**）

每个步骤独立判定：

1. **硬性下限**：反对加权总分 **≥ 6** → **立即否决**（无需等截止）
   - 例：2 名相关运营者反对 = 3+3 = 6
   - 例：1 名相关运营者 + 3 名普通组员反对 = 3+1+1+1 = 6

2. **截止时**：
   - 反对加权占比 **< 40%** → **通过**
   - 反对加权占比 **≥ 40%** → **否决**

3. 截止前若未触发硬性下限，状态保持"进行中"，组员可改票

### 匿名投票

所有投票都是匿名的。系统只显示参与人数和加权结果，**不公开具体投票人**。每个用户只能看到自己投的票。

### 默认通过倾向

由于作者本人贡献了 3 分加权同意票，默认情况下需要相当数量的反对才能否决，符合"除非反对的人特别多，不然还是应该同意"的设计意图。

## 设计思路示例

| 场景 | 反对加权分 | 结果 |
|------|----------|------|
| 5 名普通组员反对，无运营者反对 | 5 | 截止时按比例判（vs 作者 3 分），如无人同意则 5/(5+3) = 62% > 40% → 否决 |
| **组长 + 1 名相关运营者反对** | **6** | **立即否决**（即使其他人都同意） |
| 1 名相关运营者反对 + 3 名普通组员反对 | 6 | 立即否决 |
| 仅 1 名相关运营者反对（无组长） | 3 | 与作者 3 分相抵，3/(3+3)=50% > 40% → 截止时否决（除非有人投同意） |
| 3 名普通组员反对，2 名普通组员同意 | 3 | 反对占比 3/(3+3+2)=37.5% < 40% → 通过 |

## 已知限制

- **localStorage 数据隔离**（见上文）
- **客户端密码**：`config.js` 在浏览器可见，密码是明文。这只能算"轻度访问控制"，不能防有心人。如需安全请上后端。
- **时区**：使用浏览器本地时区
- **作者改票**：作者投票自动算同意，无法投反对（这是设计选择，不是 bug）
